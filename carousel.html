3 Things That Surprised Me
Surprise #1
The slowest entry is a query builder — not an ORM.
Drizzle — a "lightweight" query builder — is slower than Sequelize (a full ORM from 2014) in every category.
Surprise #2
Query builders can't beat a well-designed ORM.
Knex and Kysely have zero entity overhead. Yet UQL — a full ORM with relations and migrations — beats both in all 8 categories.
Surprise #3
MikroORM v7 improved, but complex queries remain expensive.
v7 dropped Knex and builds SQL directly — simple ops doubled in speed. But complex queries still show heavy abstraction cost: 23x slower than UQL.
6/9
Why Is UQL Faster?
The Problem
Most ORMs figure out your schema on every query: "What table? What column? Nullable?" — answered fresh each time.
UQL's Approach
Answer everything once at startup. Field→column mappings, table names, relations — all pre-computed into lookup tables.
UQL
Pre-computed metadata
→ direct buffer writes
→ zero allocation
Others
Build query object
→ walk expression tree
→ produce SQL string

What's Measured
Pure SQL string generation — no database, no I/O, no connection pool. Isolates ORM overhead only.
🚫
Why No Prisma?
Prisma is not a pure TypeScript/JavaScript ORM — only its client is JS/TS. The query engine is a separate Rust binary that builds SQL, making it architecturally incomparable. It is not even testeable this way.
